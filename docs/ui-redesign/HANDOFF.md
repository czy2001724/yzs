## 一、落地清单（按文件）

### 1. `gui/theme.py` —— 从暗色 QSS 换成亮色 token
把 `DARK_QSS` 旁边新增一套 `LIGHT_QSS`（或直接替换），色值见下表。保留 `Fusion` 基底与 `app.setStyleSheet(...)` 的用法。

| paper | 窗口底色（带暖） | `#FBF9F5` | `#13151B` |
| surface | 面板 / 卡片 | `#FFFFFF` | `#1A1D25` |
| sunk | 输入框 / 进度槽底 | `#F4F1EA` | `#14161C` |

`"Consolas","SF Mono"` 并开 `tabular-nums`。

### 2. `gui/main_window.py` —— 三栏重排 + 序号引导 + 折叠高级
- 三栏顶部各加 `①/②/③` 序号标题：**① 选一个动作 · ② 安排顺序 · ③ 运行**。
- **左栏动作库**：默认只显示 6 个常用动作瓦片（点一下 / 连点 / 输入文字 / 快捷键 / 等一会儿 / 找到图片再点），其余（移动 / 拖拽 / 滚轮 / 截图）收进「更多动作 ▾」可展开区。
- **中栏步骤**：`QListWidget` 的每项用人话渲染，例如 `1. 在 (820, 460) 点一下`、`3. 输入文字 "你好呀~"`。工具条文案：`上移 / 下移 / 修改 / 删除`，`💾 存成模板 / 📂 打开模板`。**空状态直接画"① 选动作 → ② 排顺序 → ③ 运行"三步引导。**
- **右栏运行**：大绿键 `▶ 开始运行` + `⏸`/`⏹`；`重复次数` 步进器，下方小字 `填 0 = 一直重复，直到你按停止`；进度条。
- **把"拟人化 / 反作弊 / 实时取色 / 录制"全部折叠进右下角「⚙ 高级设置」**（`QToolButton` + `setCheckable` 控制一个可折叠容器），默认收起。危险项加红色 `有风险` 小标签。
- 保留全局热键 F9/F10/F11 与"甩到左上角急停"。

### 3. `gui/step_dialogs.py` —— 少填空、多点选
- 凡是需要坐标 `x, y` 的对话框，加一个大按钮 **「🎯 去屏幕上点一下」**：点击后隐藏窗口、进入取点模式，用户在屏幕上一点即回填坐标（复用现有实时坐标/取色逻辑）。不要求用户手输 x, y。
- 术语一律按下表改文案。

### 4. `automation/engine.py` —— 仅改展示名，不动逻辑
`STEP_TYPES` 的**显示名**按下表改；`type` 键（`move/click/type/hotkey/find_image`…）保持不变以兼容已存 JSON 工作流。

## 三、设计原则（改动时的判断依据）
1. **先能跑，再谈高级** —— 第一屏只留"选动作 → 排顺序 → 运行"主线，高级项默认折叠。
2. **说人话** —— 从"用户想干嘛"命名，不用系统术语与英文动词。
3. **手把手** —— 空界面即教程；首启动弹 3 步轻引导后不再出现。
4. **少填空** —— 能点选就不让人打字（坐标用"去屏幕点"）。
5. **亮色友好** —— 暖白纸面 + 钴蓝，圆角留白；红色只留给危险操作。
