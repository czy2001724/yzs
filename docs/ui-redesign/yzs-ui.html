设计语言提案 · yzs 自动化面板

把"外挂面板",
改成谁都会用的小助手

同一个引擎,换一套说人话、有引导、亮色友好的界面语言。目标只有一个:新人打开就知道下一步点哪儿。

暗色 · 高密度 · 极客 → 亮色 · 留白 · 手把手
01

先看新人卡在哪

现有界面功能很全,但对第一次打开的人不友好。四个真实痛点,决定了整套改法。

信息过载

一开门全铺开

动作库、步骤、日志、实时状态、拟人化、录制……六块面板同时挤出来,没人告诉你"第一步该干嘛"。

术语劝退

看不懂的词太多

"拟人化""贝塞尔偏移""overshoot 越冲""GDI 读屏触发反作弊"——新人一个都不认识,直接放弃。

要手填

坐标从哪来?

加一个"点击"要手动填 x, y。新人根本不知道屏幕坐标是什么、怎么取。

压迫感

像"外挂工具"

深黑配霓虹渐变、满屏"反作弊"字样,气质偏灰产,吓退只想做点重复操作的普通用户。

02

五条设计原则

整套 UI 语言的地基。后面每一个颜色、每一句文案、每一处布局,都从这五条推导出来。

渐进呈现

先能跑,再谈高级

默认只露 6 个常用动作 + 一个大"运行"键。贝塞尔、反作弊、录制统统收进"⚙ 高级设置",想用再展开。

说人话

从"想干嘛"命名

不写系统术语,写用户目标:"组合键"→"快捷键","查找图像"→"找到图片再操作"。

手把手

空界面就是教程

没步骤时,中间直接显示"① 选动作 ② 排顺序 ③ 点运行"。第一次打开还有 3 步引导。

少填空

能点就别让人打字

坐标不手填,给一个大按钮"🎯 去屏幕上点一下",点完自动回填。

友好基调

亮色、圆角、暖中性色

浅色纸面 + 沉稳钴蓝,大量留白,柔和圆角。像一款正经的大众软件,而不是灰产工具。危险操作才用红色,给足心理安全感。

04

术语 → 人话对照表

这是新人友好度提升最大的一环。每个词都从"用户看得懂"出发,该藏风险的地方也说清楚。

旧文案(系统术语)新文案(人话)为什么
拟人化 · 反作弊像真人一样操作用户懂的是"像真人",不是"拟人化"这个抽象词
贝塞尔偏移鼠标走点弯路(更自然)数学名词无意义,描述可见效果即可
overshoot 越冲量冲过头再回来一点大白话描述动作本身
点击抖动落点轻微偏移"抖动"含糊,"落点偏移"更具体
实时像素取色(GDI 读屏)实时读屏取色 有风险先说功能,再明确标风险,不用技术缩写吓人
组合键 / hotkey快捷键(如 Ctrl+C)配上例子,一眼就懂
查找图像 / find_image找到图片再操作说清"先认画面、再动手"的因果
模板图 / template目标图片"模板"是程序员视角,"目标"是用户视角
循环次数 0 = 无限重复次数(0 = 一直重复)"循环"改"重复",并把规则写在旁边
type 输入文本 · move 移动输入文字 · 移动鼠标去掉英文动词,补全宾语

07

第一次打开:3 步上手

首启动弹一层轻引导,把"选动作 → 排顺序 → 运行"这条主线走一遍,之后不再出现。

1

从左边挑个动作

比如"点一下"。想不好就先加"输入文字",随时能改能删。

2

需要坐标?去屏幕点

点"🎯 去屏幕上点一下",到目标位置一点,坐标自动填好——不用背 x, y。

3

按 F9 或"开始运行"

鼠标甩到屏幕左上角随时急停。想更安全,再去"⚙ 高级设置"打开"像真人操作"。
